# プロジェクト管理シート（チーム用スマホアプリ）

6人チームの各自タスクを **概要 / 工程表 / メンバー別** の3画面で管理する、モバイル向けのチーム用アプリです。ホーム画面に追加すればアプリのように起動でき（PWA・オフライン対応）、Firebase を設定すると**全員で同じボードをリアルタイム共有**できます。

- 公開URL（GitHub Pages）: `https://ttomoi-afk.github.io/desktop-tutorial/pms/`
- デザイン: BROADSHEET（明朝・新聞組版。シアン＝進行中 / マゼンタ＝レビュー中 / 墨＝完了 / 破線＝未着手）
- ビルド不要の静的アプリ（HTML / CSS / Vanilla JS のみ、フレームワークなし）

## できること

- **タスクの追加・編集・削除**（右下の＋ボタン、または各行をタップ）。案件・担当・開始日・期限・ステータス・進捗率を編集。
- **概要**：全体進捗、案件別進捗、メンバー別の負荷（担当件数・平均進捗）。
- **工程表**：開始日〜期限から自動で描くガントチャート。マゼンタ線＝今日。
- **メンバー別**：メンバーを選んで担当タスク一覧。
- **メンバー / 案件の管理**、**表示項目の切替**（最終目標 / 期限 / 進捗率 / ステータス）は設定から。
- **エクスポート / インポート**（JSON）、**サンプルに戻す**。

## ホーム画面に追加（インストール）

- **iPhone (Safari)**：共有ボタン →「ホーム画面に追加」。
- **Android (Chrome)**：メニュー →「アプリをインストール」/「ホーム画面に追加」。

スタンドアロン（フルスクリーン）で起動し、オフラインでも開けます。

## クラウド同期のセットアップ（チームで共有する）

既定では**この端末内のみ**にデータを保存します（＝ローカルモード）。次の手順で Firebase を設定すると、チームでリアルタイム共有できます。所要 5分ほど・無料枠で十分です。

1. [Firebase コンソール](https://console.firebase.google.com/) で**プロジェクトを作成**。
2. **Realtime Database** を作成（Build → Realtime Database → データベースを作成）。ロケーションを選び、いったん「ロックモード」で開始で構いません。
3. **ルール**を次の内容に設定（Realtime Database → ルール → 公開）。ログイン済み（匿名）だけが読み書きでき、チームコードを知っている人だけがそのボードに入れます。
   ```json
   {
     "rules": {
       "boards": {
         "$code": { ".read": "auth != null", ".write": "auth != null" }
       }
     }
   }
   ```
4. **匿名ログインを有効化**（Authentication → Sign-in method → 匿名 → 有効）。
5. **ウェブアプリを登録**して構成をコピー（プロジェクトの設定 → マイアプリ → `</>` ウェブ）。`apiKey` や `databaseURL` などが表示されます。
6. コピーした値で **`firebase-config.js`** の `YOUR_…` を置き換え、コミット／デプロイ。
   （この構成はクライアントに埋め込む前提の公開情報で、秘密鍵ではありません。アクセスは上のルールで守ります。）
7. アプリを開き、**設定 → チーム接続**で**チームコード（合言葉）**を入力して「接続」。同じコードを入れたメンバー同士で同じボードを共有します。バッジが「クラウド同期」になれば接続成功です。

> セキュリティ：チームコードは推測されにくい文字列にし、社内など信頼できる範囲で共有してください（機微情報の保存には向きません）。より厳密にするなら、ルールにメンバー管理を足すか、メール認証への切替を検討してください。

うまく接続できないときは、ルール未設定 / 匿名ログイン無効 / `databaseURL` の誤り、が典型です。接続に失敗してもアプリはローカルモードで動き続けます。

## デプロイ（GitHub Pages）

このリポジトリは `main` の更新で GitHub Pages に自動デプロイされます。`pms/` 以下をそのまま配信するだけで動作します（Service Worker のスコープは `pms/` に限定され、同居する他ページには干渉しません）。

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | アプリの外枠（3画面・タブ・各エディタ） |
| `styles.css` | BROADSHEET トークンと画面スタイル |
| `store.js` | データモデル・ローカル保存・集計（全体46%やガント位置の算出） |
| `sync.js` | 同期層。`FirebaseBackend`（本番）/ `LocalBackend`（端末内・タブ間） |

| `firebase-config.js` | Firebase 構成（クラウド同期を使うとき記入） |
| `manifest.webmanifest`, `sw.js`, `icon-*.png` | PWA（ホーム追加・オフライン） |
| `notify/reminders.mjs` | 期限通知の送信スクリプト（GitHub Actions から実行） |
| `../.github/workflows/task-reminders.yml` | 毎日の通知スケジュール |

## タスク期限のメール通知（GitHub Actions）

**毎朝 8:00 JST**に、その日が期限のタスク（＋期限を過ぎた未完了タスク）を、**担当者本人のメールアドレス**へ送ります。対象タスクがある人にだけ届きます。GitHub Actions のスケジュールで動くため**無料**、Firebase も無料プランのまま（匿名読み取りで参照）で動きます。

**有効化の手順**

1. **アプリでメンバーにメールアドレスを登録**：設定 →（メンバーの）編集 → メールアドレス。
2. **GitHub にシークレットを登録**：リポジトリ → Settings → Secrets and variables → Actions → *New repository secret* で以下を作成。
   | 名前 | 値 |
   | --- | --- |
   | `PMS_TEAM_CODE` | アプリで使っているチームコード（合言葉） |
   | `SMTP_HOST` | 例）`smtp.gmail.com` |
   | `SMTP_PORT` | 例）`465` |
   | `SMTP_USER` | 送信元メールアドレス |
   | `SMTP_PASS` | SMTP パスワード（下記） |
   | `MAIL_FROM` | 差出人に表示するメールアドレス（通常 `SMTP_USER` と同じ） |
3. **SMTP パスワードの用意**
   - **Gmail / Google Workspace**：Google アカウント → セキュリティ → 2段階認証を有効化 → **アプリ パスワード**を生成し、それを `SMTP_PASS` に。`SMTP_HOST=smtp.gmail.com` / `SMTP_PORT=465`。
     （Workspace 側でアプリ パスワードが禁止されている場合は下の代替をご利用ください。）
   - **代替（Resend 等の無料メールAPI）**：`SMTP_HOST=smtp.resend.com` / `SMTP_PORT=465` / `SMTP_USER=resend` / `SMTP_PASS=<APIキー>` / `MAIL_FROM=<検証済みドメインのアドレス>`。
4. **手動テスト**：Actions タブ → *Task deadline reminders* → **Run workflow** → `dry_run` に `true` を入れて実行。送信せずに「誰に何を送るか」をログで確認できます。問題なければ翌朝の自動送信を待つ（または `dry_run=false` で即送信）。

**補足・注意**

- スケジュールは UTC 基準（8:00 JST = 23:00 UTC）。GitHub の混雑時は数分〜十数分ずれることがあります。
- **リポジトリが 60 日間更新されないと、GitHub の仕様でスケジュール実行が自動停止**します（その後に何かコミット/手動実行すれば再開）。
- シークレット未設定のうちは、ジョブは「未設定のためスキップ」して**正常終了**します（赤いエラーにはなりません）。
- メール未登録のメンバーは送信対象外（実行ログに一覧表示）。

## 注意

- 収録データは**ダミー**です。お名前・案件・タスクは実運用に合わせて編集してください。
- クラウド未設定でもすべての機能はローカルで使えます（共有のみ Firebase 設定が必要）。
